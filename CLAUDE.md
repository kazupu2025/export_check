## Git 運用ルール

### コード変更のたびに必ず push する

コードを変更・追加・削除したら、**作業単位ごとに必ず以下を実行すること**：

```sh
git add <変更ファイル>
git commit -m "説明的なコミットメッセージ"
git push origin <ブランチ名>
```

- `git add .` や `git add -A` は使わず、変更ファイルを明示的に指定する（.env などの誤コミット防止）
- コミットメッセージは「何をなぜ変えたか」を日本語または英語で簡潔に書く
- push は省略しない。ローカルにのみ留めない

### ブランチ戦略

- `main` ブランチへの直接 push は原則禁止
- 機能追加・修正は `feature/xxx` または `fix/xxx` ブランチで作業
- 作業完了後は PR を作成してレビュー → マージ

### コミット前チェック

- `.env` ファイルや API キーを含むファイルがステージに含まれていないか確認
- `git status` で意図しないファイルがないか確認してからコミット

## セキュリティルール

- API キー・URL・パスワードはコードに直書きしない
- 環境変数は `.env` ファイルで管理し、`.gitignore` に追加する
- ドキュメント内のシークレットはすべて `<your-key>` 形式のプレースホルダーにする

## 開発スタイル

- コメントは「なぜ」が自明でない場合のみ書く（「何をしているか」は書かない）
- 不要な抽象化・将来の要件への対応は行わない
- エラーハンドリングはシステム境界（ユーザー入力・外部 API）にのみ追加する
